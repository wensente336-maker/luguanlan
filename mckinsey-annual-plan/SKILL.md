# 麦肯锡年度规划 PDCA 助手

## 概述

本 Skill 以 PDCA 循环为核心框架，整合全套麦肯锡经典管理工具，引导用户完成从年度战略规划、目标拆解、执行落地到复盘优化的完整闭环。支持「全流程从头做」和「单阶段切入」两种模式，适用于公司级、部门级、项目级的各类计划制定与迭代场景。

## 阶段判断：从哪里开始

根据用户需求判断当前所处阶段，按需切入：

| 用户表述关键词 | 切入阶段 | 核心产出 |
|---------------|----------|----------|
| 制定年度计划、定目标、做规划、战略规划 | P 计划 | 年度战略目标 + KPI 拆解树 |
| 拆解目标、落任务、做行动计划、排期 | D 执行 | 行动计划表 + 里程碑节点 |
| 复盘、检查进度、季度总结、找差距 | C 检查 | 差距分析报告 + 根因诊断 |
| 优化、调整、改进、年度总结、下一年规划 | A 优化 | 改进方案 + 下一轮规划输入 |

若用户未明确阶段，默认从 P 阶段开始走完整 PDCA 流程。

## 【P 阶段】计划 Plan：年度战略规划与目标设定

### 执行步骤

1. **外部环境扫描**
   - 用 PEST 分析宏观环境（政治、经济、社会、技术）
   - 用波特五力分析行业竞争格局
   - 工具详情：`references/phase-plan.md`

2. **内部能力盘点**
   - 用 SWOT 做内外部综合分析
   - 用 7S 模型诊断组织健康度（战略与能力是否匹配）
   - 工具详情：`references/phase-plan.md`

3. **业务战略选型**
   - 用 3C 分析（公司-客户-竞品）找准定位
   - 用安索夫矩阵选择增长路径（市场渗透/开发/产品开发/多元化）
   - 用波士顿矩阵优化业务组合（明星/金牛/问题/瘦狗）
   - 工具详情：`references/phase-plan.md`

4. **年度总目标设定（KGI）**
   - 确定 3-5 个顶层关键目标，符合 SMART 原则
   - 财务目标 + 业务目标 + 组织目标 三维平衡

5. **目标层层拆解（KPI 树）**
   - 公司级 KGI → 部门级 KPI → 岗位级 KDI
   - 遵循 MECE 原则：不重复、不遗漏
   - 工具详情：`references/phase-plan.md`

### 阶段产出
- 年度战略目标表
- KPI 拆解树
- 模板：`references/templates.md`

## 【D 阶段】执行 Do：行动计划拆解与落地

### 执行步骤

1. **任务拆解（逻辑树 How 树）**
   - 每个 KPI 反向拆解为具体行动项
   - 确保每个动作都对指标有直接贡献

2. **5W1H 要素补全**
   - 每项任务明确：谁（Who）、做什么（What）、什么时候（When）、在哪（Where）、怎么做（How）、预算多少（How much）

3. **优先级排序**
   - 时间四象限：重要/紧急 二维分类
   - 决策矩阵：效果/成本/可行性 综合打分
   - 工具详情：`references/phase-do.md`

4. **里程碑设置**
   - 按季度/月度划分检查节点
   - 每个节点设定明确的交付物和衡量标准

5. **风险预判**
   - 识别关键风险点，制定应对预案
   - 工具详情：`references/phase-do.md`

### 阶段产出
- 年度行动计划表
- 里程碑节点表
- 模板：`references/templates.md`

## 【C 阶段】检查 Check：进度复盘与差距分析

### 执行步骤

1. **数据对标**
   - 实际结果 vs 计划目标，逐项计算完成率
   - 按指标分类统计：超额完成、基本达标、严重偏差

2. **差距分析**
   - 量化偏差绝对值和相对值
   - 区分正向偏差（超预期）和负向偏差（未达标）
   - 工具详情：`references/phase-check.md`

3. **根因深挖**
   - 用 5Why 连续追问找根本原因
   - 用逻辑树（Why 树）系统拆解原因
   - 三类归因：目标设定问题 / 执行不到位 / 外部环境变化
   - 工具详情：`references/phase-check.md`

4. **分类总结**
   - 成功经验项：可复制的做法
   - 问题项：需要纠正的偏差
   - 风险项：需要关注的潜在问题

### 阶段产出
- 季度/年度复盘报告
- 差距分析表
- 模板：`references/templates.md`

---

## 【A 阶段】优化 Act：经验固化与迭代改进

### 执行步骤

1. **成功经验标准化**
   - 验证有效的方法固化为 SOP / 标准流程
   - 纳入下一年度的基准操作规范

2. **问题项制定纠正措施**
   - 每个问题对应具体改进动作
   - 明确责任人和完成期限
   - 工具详情：`references/phase-act.md`

3. **未完成事项流转**
   - 评估未完成事项的优先级
   - 转入下一轮 PDCA 循环重新规划

4. **下一轮规划输入**
   - 输出改进建议和经验教训
   - 作为下一周期 P 阶段的重要输入
   - 形成「年度大循环 + 季度小循环 + 月度微循环」的滚动迭代机制

### 阶段产出
- 优化改进方案
- 下一期规划建议
- 模板：`references/templates.md`

## 输出规范

1. 默认输出结构化表格 + 要点总结，清晰易读
2. 所有目标和指标必须可衡量，避免模糊表述
3. 目标拆解遵循 MECE 原则（相互独立、完全穷尽）
4. 用户要求"生成文档"时，整理为完整文档格式输出
5. 需要参考完整案例时，查看 `references/examples.md`
